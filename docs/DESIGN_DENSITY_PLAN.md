## Зачем нужен этот план

Этот файл фиксирует отдельный проход по визуальной плотности интерфейса.

Проблема текущей версии:

- некоторые блоки занимают слишком много вертикального пространства при малом объёме информации;
- карточки `Services` выглядят крупнее, чем требует их содержимое;
- `Audience` воспринимается лучше, но всё ещё имеет избыточный воздух;
- общий ритм секций местами выглядит растянутым, а не собранным.

## Цель

Сделать интерфейс визуально плотнее и спокойнее, не ломая:

- коричневый фон `#3b3330`;
- оранжевый акцент `#ff7600`;
- hero-композицию;
- блок брендов и нижнюю details-панель;
- адаптив и текущую сеточную логику сайта.

## Принцип

Плотность должна зависеть от объёма информации:

- мало текста -> компактнее карточка и меньше vertical spacing;
- много текста -> больше воздуха допустимо;
- не все секции должны выглядеть одинаково крупными.

## Фазы

### Phase 0. Аудит

Проверить:

- общий контейнер;
- секционные `padding`;
- размеры заголовков;
- плотность карточек;
- поведение на desktop/tablet/mobile.

### Phase 1. Локальное уплотнение Services + Audience

Цель:

- не трогать hero, brands, equipment, about;
- уменьшить визуальную тяжесть только у двух проблемных секций.

Изменения:

- уменьшить vertical padding секций;
- уменьшить локальную ширину контентной области;
- уменьшить высоту карточек `Services`;
- уменьшить размеры номера, типографики и внутренних отступов;
- слегка собрать `Audience`, не делая его мелким.

Критерии успеха:

- `Services` перестаёт ощущаться как длинная стена из oversized-карточек;
- `Audience` выглядит собраннее;
- desktop не ломается;
- mobile остаётся читаемым.

### Phase 2. Ритм секций

Цель:

- выровнять ощущение масштаба между соседними секциями.

Потенциальные действия:

- откалибровать `h2` по секциям;
- проверить зазоры между `Intro -> Audience -> Services -> Brands`;
- убрать лишний воздух там, где контент утилитарный.

Эта фаза делается только после проверки Phase 1.

### Phase 3. Финальная шлифовка

Проверить:

- desktop screenshot;
- tablet screenshot;
- mobile screenshot;
- скролл-ощущение страницы;
- нет ли новых конфликтов по адаптиву.

## Что не делать без отдельной задачи

- не переделывать дизайн сайта заново;
- не менять структуру hero;
- не перестраивать brands;
- не менять тексты;
- не менять палитру.

## Текущее решение

На ближайшем проходе приоритет отдан `Phase 1`: сначала локально уплотняем `Services` и `Audience`, потом проверяем результат и только после этого решаем, нужен ли более широкий ритм-пасс по всему лендингу.
