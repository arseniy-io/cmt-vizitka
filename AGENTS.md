# AGENTS.md

Этот файл нужно читать первым при работе с `cmt-landing`. Он даёт короткий контекст и направляет к нужному документу, чтобы не перечитывать весь проект.

## Что это за проект

`cmt-landing` - отдельный React 19 + Vite лендинг Центра Мебельных Технологий. Это B2B-сайт-визитка о производстве мебели под маркой клиента.

Проект не связан с Mini App, Tilda, padel-клубом, кортами, календарём бронирования или оплатой.

## Главные правила

- Не добавлять новые секции, тексты и функции без прямой просьбы пользователя.
- Не возвращать ничего про padel, корты и бронирование.
- Не использовать код или структуру других проектов.
- Не менять дизайн всего сайта ради локальной правки.
- Каждое визуальное изменение проверять на desktop, tablet и mobile.
- Активные стили находятся только в `src/styles.css`.
- Не подключать UI-библиотеки без необходимости и согласования.
- Не откатывать существующие изменения пользователя.
- Не добавлять в Git `dist`, `node_modules`, логи, скриншоты и результаты тестов.

## Поведение, которое важно сохранить

- На мобильной шапке меню скрыто, остаются логотип и кликабельные телефоны.
- В услугах 10 уникальных карточек. Одновременно раскрывается только одна.
- Карточки брендов не растут. Подробности открываются отдельной панелью под всей сеткой.
- Видео оборудования не запускаются сами. Пользователь включает одно видео вручную, начальная громкость - 10%.
- Форма локальная и не отправляет данные на сервер.

Подробности и причины этих решений находятся в [docs/DECISIONS.md](docs/DECISIONS.md).

## Обычный порядок работы

1. Проверить `git status`.
2. Открыть только нужный компонент и связанный раздел `src/styles.css`.
3. Внести точечную правку.
4. Запустить `npm run lint` и `npm run build`.
5. Для визуальных изменений запустить сайт и `npm run screenshot`.
6. Если изменилось устройство проекта, обновить связанный документ.

Последняя просьба пользователя всегда важнее документации. Если код и документ расходятся, проверить реальное поведение, исправить документ и не угадывать.
